Glossary

User​

A User represents a profile of a user. User has to create a profile with their name, email, and metadata.

Group​

A Group represents a group of users. There are predefined roles for a user like Team Admin and Team Members. By default, every user gets a Team member role. Team admin can add and remove members from the Group.

Role​

Within a namespace, a role will get assigned to users. A role is needed to define policy. There are predefined roles in a namespace. Users can also create custom roles.

Namespace​

Namespace provides scope for the policies. There are predefined namespaces like organization, project, group, etc. Users can also create custom namespaces.

Policy​

A Policy defines what actions a role can perform in a namespace. All policies in a namespace will be used to generate schema for spicedb.

Action​

Within a namespace, a role can perform certain actions. A action is needed to define policy.

Project​

A Project is a scope in which a User can create and manage Resources.

Orgnaization​

An Orgnaization is just a group of Projects. Groups also belongs to Orgnaization.

Resource​

Resources are just custom namespaces in which user can define policies. All Resources also gets default policies.

Spicedb​

SpiceDB is a Zanzibar-inspired open source database system for managing security-critical application permissions.
